The error you're seeing is due to a compatibility issue between the `better-sqlite3` native Node.js module and the Alpine Linux base image used in your Dockerfile. Alpine uses musl libc, which doesn't fully support the pre-built binaries for `better-sqlite3`, leading to the "fcntl64: symbol not found" error. This prevents the database from initializing in your application.

To fix this, I'll update the Dockerfile to use a standard Debian-based Node.js image (`node:18`) instead of Alpine. This avoids the musl libc issues while keeping the image lightweight and compatible with native modules like `better-sqlite3`. The `docker-compose.yml` file doesn't need changes, as the issue is in the build process.

Here's the updated Dockerfile:

Dockerfile
```
# Use Node.js 18 for better compatibility with native modules like better-sqlite3
FROM node:18 AS base

# Install dependencies for SQLite (Debian-based)
RUN apt-get update && apt-get install -y sqlite3 libsqlite3-dev && rm -rf /var/lib/apt/lists/*

# Set working directory
WORKDIR /app

# Copy package files
COPY package*.json ./

# Install dependencies
RUN npm ci --only=production

# Production stage
FROM base AS production

# Copy installed dependencies from base stage
COPY --from=base /app/node_modules ./node_modules

# Copy application source
COPY . .

# Create directory for SQLite database
RUN mkdir -p /app/data

# Set database path environment variable
ENV DATABASE_PATH=/app/data/ledger.db

# Expose port
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "fetch('http://localhost:3000/api/health').then(r => r.ok ? process.exit(0) : process.exit(1))"

# Start the application
CMD ["npm", "start"]
```
